AI Receptionist for a Dental Clinic

My role: Sole builder — requirements, architecture, implementation, tests.

Python 3.12 Claude API Finite-State Machine Business-Rule Engine YAML-driven config 63 tests
📱
Add screenshot / GIF here Suggestion: a short screen recording of the console chat booking a first visit and refusing an invalid slot.

Problem

A clinic with 15 years of operation and zero automation was losing first-visit bookings: no one could consistently answer, triage urgencies and schedule around complex rules (a doctor who rotates between two countries, orthodontics only on Wednesdays, booking windows, legal consent).

What I built

A conversational AI receptionist that answers FAQs, books first valuations, detects emergencies and escalates them, and enforces every business rule of the clinic—running fully in mock mode at zero API cost, then switching to the real model with no code change.

Architecture & stack

Python (managed with uv), cleanly modularized: an FSM for conversation state, an intent detector with a heuristic fallback, a separate rule engine, YAML knowledge files editable without touching code, and a dual MOCK/real-API LLM wrapper. A consent flow implements Colombia's data-protection law (Habeas Data).

The hard part & how I solved it

Stopping the LLM from "hallucinating" appointments. My fix: the business rules live in code, not in the prompt. The model only adds natural language; the code decides if, when and at what time a booking is valid. Emergencies always escalate to a human—never auto-booked.

Result

A working end-to-end MVP with 63 passing tests, validated in mock mode (US$0) and ready to run against the live model by adding one API key. Designed as the validation step before connecting WhatsApp and the clinic's scheduling software.

AI Process-Mapping SaaS for SMBs

My role: Sole builder — product design, architecture, methodology, full-stack build.

React + Vite Tailwind Node / Express SQLite JWT + bcrypt Claude API (structured output) SVG visualization
🗺️

Problem

Small businesses can't afford traditional operations consultants, yet they have no clear view of where they lose time and money—bottlenecks, waits and rework hide inside everyday processes.

What I built

A full web app that turns what employees describe into a process map (swimlane diagram) with bottlenecks, waits, rework, metrics, insights and prioritized recommendations—grounded in real consulting methodology, not generic AI output.

Architecture & stack

React + Vite + Tailwind front end; Node/Express back end; SQLite; multi-company accounts with JWT + bcrypt auth and an admin panel. The Claude API key lives only on the back end behind a secure proxy—the piece that makes it a sellable product, not a toy.

The hard part & how I solved it

Making AI output trustworthy and consistently structured. I grounded the prompts in established methodologies (SIPOC, Lean, Theory of Constraints, DMAIC) and used structured outputs to guarantee the format, then rendered the result as an interactive SVG swimlane with click-through detail.

Result

Works end to end locally in both dev and production builds, with a seeded demo company and PDF export. A "demo mode" runs the whole experience with no API key, so anyone can try it instantly. Screenshots verified across the full flow.
